## Documentation Standards

All LaunchWatch documentation follows these principles:

1. **Simple** - Easy to understand, no jargon
2. **Accurate** - Reflects actual codebase
3. **Up-to-date** - Updated with each release
4. **Action-oriented** - Tells you what to do
5. **Examples** - Shows, not just tells

## Updating Documentation

When making changes that affect documentation:

1. Update relevant docs in the same PR
2. Check all cross-references still work
3. Update CHANGELOG.md
4. Add to PR checklist

## Missing Something?

If documentation is unclear or missing:

1. [Open an issue](https://github.com/yourusername/launchwatch/issues/new)
2. Label it `documentation`
3. Describe what's needed

Or better yet, submit a PR to improve it!
